The project combines numerical methods, data handling, and economic reasoning to produce transparent and reproducible results across three connected sub-projects.

## Project Overview
The objective of this repository is to document and implement key tools from introductory scientific programming and numerical analysis in an economics context.  
Each project is designed to highlight a different part of the pipeline:
- mathematical and computational foundations,
- empirical data collection and analysis,
- structured economic modeling and simulation.

## Repository Structure
The repository is organized into dedicated project folders for:
- `inaugural project`
- `data project`
- `model project`

Each project contains code, outputs, and written discussion of assumptions, method choices, and conclusions.

## Model Project
The model project develops and analyzes an economic model to study ...  
It emphasizes parameter choices, computational solution strategy, and interpretation of model outcomes.

## Methodological Workflow
Across all projects, the workflow follows a common structure:
1. Define the economic question and assumptions.
2. Implement numerical and/or statistical methods in code.
3. Validate intermediate results and assess robustness.
4. Present findings with clear interpretation and limitations.

## Reproducibility
To support reproducibility, the repository is structured to keep code and analysis traceable.  
When running the projects, use consistent package versions and execute notebooks/scripts in their intended order within each project folder.
